# 테스트 병령어 정리

이 문서는 Docker dev container 내부에서 바로 실행할 수 있는 테스트 명령어를 정리한 문서다.

- 기준 작업 경로: `/workspaces/B_PLUS-TREE`
- 기준 테이블: `users`
- 기준 전제: `schema/users.schema`, `data/users.dat`, `sqlp`, `sqlp_sim`, `test_perf`, `test_perf_sim`가 존재함

## 1. 기본 준비

```bash
cd /workspaces/B_PLUS-TREE
```

빌드부터 다시 확인하고 싶으면 아래를 순서대로 실행한다.

```bash

make sim
make perf
make perf_sim
bash scripts/lint.sh
```

## 3. 가장 기본적인 SELECT 확인

## 6. compare 모드 확인

`--compare`는 같은 `SELECT`를 auto 경로와 forced-linear 경로로 각각 한 번씩 실행한다.

- `stdout`: `[AUTO RESULT]`, `[LINEAR RESULT]` 두 결과 테이블
- `stderr`: 비교 요약 1줄

### 6-1. `id = 5000` compare

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_compare_id_eq.sql <<'EOF'
SELECT * FROM users WHERE id = 5000;
EOF

./sqlp --compare /tmp/q_compare_id_eq.sql 2>&1
```

### 6-2. compare 결과를 stdout / stderr로 분리해서 보기

```bash
cd /workspaces/B_PLUS-TREE

./sqlp --compare /tmp/q_compare_id_eq.sql >/tmp/compare_out.txt 2>/tmp/compare_err.txt

echo "[stdout]"
cat /tmp/compare_out.txt

echo "[stderr]"
cat /tmp/compare_err.txt
```

확인 포인트:

- `stdout`에 `[AUTO RESULT]`, `[LINEAR RESULT]` 둘 다 있는지
- `stderr`에 `[COMPARE] auto=... linear=... speedup=...` 한 줄이 있는지

### 6-3. `id BETWEEN` compare

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_compare_id_between.sql <<'EOF'
```

### 6-4. `age BETWEEN` compare

## 7. sim 빌드에서 tree_io 확인

- 인덱스 경로: `tree_io > 0`
- linear 경로: `tree_io = 0`

### 7-1. `id = 5000` sim 자동 경로

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_sim_id_eq.sql <<'EOF'
SELECT * FROM users WHERE id = 5000;
```

### 7-2. `age BETWEEN` sim 자동 경로

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_sim_age_between.sql <<'EOF'
SELECT * FROM users WHERE age BETWEEN 30 AND 40;
EOF

./sqlp_sim /tmp/q_sim_age_between.sql 2>&1
```

### 7-3. `age BETWEEN` sim 강제 linear 경로


확인 포인트:

- 자동 경로는 `tree_io`가 0보다 큰지
- linear 경로는 `tree_io=0`인지

## 8. 여러 번 반복해서 sim 차이 체감하기

한 번 실행으로는 차이가 작게 보일 수 있어서 반복 실행으로 전체 시간을 비교한다.

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_loop.sql <<'EOF'
SELECT * FROM users WHERE id = 5000;
EOF

time sh -c 'for i in $(seq 1 300); do ./sqlp /tmp/q_loop.sql >/dev/null 2>/dev/null; done'
time sh -c 'for i in $(seq 1 300); do ./sqlp_sim /tmp/q_loop.sql >/dev/null 2>/dev/null; done'
```

확인 포인트:

- `sqlp_sim` 쪽 전체 시간이 더 크면 sim 효과가 들어간 것

## 15. 확인해야 하는 핵심 체크리스트

- 기본 `sqlp`에서 `id = ...`는 `index:id:eq`로 가는지
- `--force-linear`에서 `id BETWEEN`, `age BETWEEN`이 정상 동작하는지
- `--compare`에서 auto 결과와 linear 결과가 둘 다 출력되는지
- `--compare`에서 stderr 요약이 한 줄만 나오는지
- `sqlp_sim`에서 인덱스 경로는 `tree_io > 0`, linear는 `tree_io=0`인지
- `test_perf`, `test_perf_sim`에서 4개 섹션이 모두 출력되는지
- 큰 범위 `BETWEEN`에서 결과가 잘리지 않는지
