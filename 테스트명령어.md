# 테스트 명령어 정리

이 문서는 Docker dev container 내부 `/workspaces/B_PLUS-TREE` 기준으로 바로 실행할 수 있는 명령어를 정리한 문서다.

- 기본 작업 경로: `/workspaces/B_PLUS-TREE`
- 기본 테이블: `users`
- 성능 벤치 실행 파일: `./test_perf`, `./test_perf_sim`
- SQL 실행 파일: `./sqlp`, `./sqlp_sim`

## 1. 기본 준비

```bash
cd /workspaces/B_PLUS-TREE
```

빌드가 필요하면 아래 순서로 준비한다.

```bash
make clean
make
make sim
make perf
make perf_sim
bash scripts/lint.sh
```

## 2. 벤치 데이터 준비

`data/users.dat`가 아직 없다면 먼저 데이터를 만들고 적재한다.

```bash
cd /workspaces/B_PLUS-TREE

make gen_data
./gen_data 10000 users
./sqlp samples/bench_users.sql >/tmp/load_out.txt 2>/tmp/load_err.txt

tail -n 20 /tmp/load_err.txt
```

## 3. test_perf 실행 규칙

이제 `test_perf`와 `test_perf_sim`은 세 번째 인자로 실행할 테스트를 고를 수 있다.

```bash
./test_perf [table] [rows] [all|point|id-range|id-range-large|age-range|height]
./test_perf_sim [table] [rows] [all|point|id-range|id-range-large|age-range|height]
```

케이스 의미는 아래와 같다.

- `all`: 전체 테스트 실행
- `point`: 단건 조회 `WHERE id = rows/2`
- `id-range`: 기본 범위 조회 `WHERE id BETWEEN rows/4 AND rows/2`
- `id-range-large`: 대범위 조회 `WHERE id BETWEEN 1 AND rows`
- `age-range`: `WHERE age BETWEEN 30 AND 40`
- `height`: `order=4`와 `order=128` 높이 비교

전체 실행 시 출력 순서는 아래와 같다.

- 테스트 1: 단건 조회
- 테스트 2: 범위 조회(id)
- 테스트 3: 대범위 조회(id)
- 테스트 4: 범위 조회(age)
- 테스트 5: 높이 비교

## 6. 자주 같이 보는 sqlp 명령

### 6-1. 자동 경로와 선형 스캔 비교

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_compare_id_eq.sql <<'EOF'
SELECT * FROM users WHERE id = 5000;
EOF

./sqlp --compare /tmp/q_compare_id_eq.sql 2>&1
```

### 6-2. 강제 선형 스캔 단독 실행

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_force_id_range.sql <<'EOF'
SELECT * FROM users WHERE id BETWEEN 2500 AND 5000;
EOF

./sqlp --force-linear /tmp/q_force_id_range.sql 2>&1
```

### 6-3. age 범위 비교

```bash
cd /workspaces/B_PLUS-TREE

cat > /tmp/q_compare_age_range.sql <<'EOF'
SELECT * FROM users WHERE age BETWEEN 30 AND 40;
EOF

./sqlp --compare /tmp/q_compare_age_range.sql 2>&1
```

## 7. stdout / stderr 분리 확인

조회 결과 표와 시간 로그를 분리해서 보고 싶으면 아래처럼 실행한다.

```bash
cd /workspaces/B_PLUS-TREE

./sqlp --compare /tmp/q_compare_id_eq.sql >/tmp/compare_out.txt 2>/tmp/compare_err.txt

echo "[stdout]"
cat /tmp/compare_out.txt

echo "[stderr]"
cat /tmp/compare_err.txt
```
